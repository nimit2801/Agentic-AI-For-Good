03

The problem

Discovery breaks at the
moment it matters most.

  • Developers find tools via Twitter, word of mouth, blog posts. None of which are in their IDE.
  • The average AI dev uses 3–5 new tools per project cycle
  • "Awesome lists" have 500+ entries. Nobody reads them.
  • Your tool might be exactly right for someone, but they just don't know it exists
03

The DevRel blind spot

Great documentation doesn't matter if nobody finds your tool.

Architecture

tools/*.yaml  →  GitHub Actions CI  →  Supabase (pgvector)
     ↑                                        ↓
  PR from                               Next.js on Vercel
  community                                   ↓
                                       Claude via MCP
                                  (Desktop · Code · API)
Catalog

YAML files in git. Validated on PR. Auto-synced on merge.

Search

pgvector semantic search + full-text fallback. No hallucination.

MCP

Hosted HTTP endpoint + npm package. One config line.

07

Model Context Protocol

MCP = the USB-C of AI integrations.

Anthropic's open standard. Connects external data and tools to any LLM in-context, at inference time. One config entry in Claude Desktop / Claude Code → your tool is live in every conversation. 

# Install once
npx agentic-ai-for-good-mcp

# Or hosted (no install)
https://agenticaiforgood.com/api/mcp
08

MCP tools available

Tool What it does Example
search_tools Semantic search by use case "vector DB for python agents"
get_tool_detail Full install, docs, code samples get_tool_detail("langchain")
suggest_for_stack Reads package.json / requirements.txt stack-aware gaps
whats_new Tools added in last N days whats_new(days=7)

Live demo · audience-driven

Bring your AI tool. We'll integrate it live.

  1. search_tools("agent framework with memory")
  2. get_tool_detail("langchain") for install + code sample
  3. suggest_for_stack: drop in a real package.json
